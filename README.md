# Street Food Map - Data Ingestion Pipeline

A TypeScript-based data pipeline that extracts video metadata and transcripts from Indian food vlog YouTube channels, starting with Delhi Food Walks.

## Overview

India has thousands of street food vendors documented across hundreds of YouTube food vlogs. This pipeline extracts that knowledge into a structured, searchable format.

## Features

- **Channel Resolution**: Resolve YouTube channel handles to channel IDs
- **Playlist Extraction**: Fetch all playlists from a channel
- **Video Extraction**: Fetch all videos with full metadata (tags, duration, etc.)
- **Transcript Extraction**: Download transcripts with language priority (Hindi > English)
- **Rate Limiting**: Configurable delays and exponential backoff
- **Checkpointing**: Resume interrupted runs without re-fetching data
- **Quota Tracking**: Monitor YouTube API quota usage

## Installation

```bash
# Install dependencies
npm install

# Copy environment template
cp .env.example .env

# Add your YouTube API key to .env
# Get one at: https://console.cloud.google.com/apis/credentials
```

## Usage

```bash
# Full pipeline
npm run dev -- --channel @delhifoodwalks

# Individual steps (for debugging/partial runs)
npm run dev -- --channel @delhifoodwalks --step playlists
npm run dev -- --channel @delhifoodwalks --step videos
npm run dev -- --channel @delhifoodwalks --step transcripts

# Resume interrupted run
npm run dev -- --channel @delhifoodwalks --resume

# Dry run (show what would be fetched, no API calls)
npm run dev -- --channel @delhifoodwalks --dry-run

# Verbose logging
npm run dev -- --channel @delhifoodwalks --verbose
```

## Output

Data is stored in the `data/raw/` directory:

```
data/
└── raw/

    ├── progress.json       # Checkpoint for resuming

        └── {video_id}.json # One file per video
```

## Configuration

Environment variables (`.env`):

| Variable | Default | Description |
|----------|---------|-------------|
| `YOUTUBE_API_KEY` | (required) | YouTube Data API v3 key |
| `YOUTUBE_API_DELAY` | 100 | Delay between API calls (ms) |
| `TRANSCRIPT_FETCH_DELAY` | 500 | Delay between transcript fetches (ms) |

## Error Handling

| Error | Handling |
|-------|----------|
| Invalid API key | Exit with setup instructions |
| Channel not found | Exit with error message |
| Quota exceeded | Save progress, exit with retry message |
| Network timeout | Retry 3x with exponential backoff |
| Transcript unavailable | Create error marker file, continue |
| Private/deleted video | Log and skip |

## API Quota

YouTube Data API has a daily quota of 10,000 units. Cost per operation:
- Search: 100 units
- Channels/Playlists/Videos list: 1 unit each

For a channel with 500 videos:
- Channel resolution: ~100 units
- Playlist fetch: ~10 units
- Video fetch: ~50-100 units
- Video enrichment: ~10 units

Total: ~200-300 units per full run (well within daily limit).

Transcript fetching uses a separate library (no API quota).
